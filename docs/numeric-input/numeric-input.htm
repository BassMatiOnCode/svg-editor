Numeric Input Controls

Harmonizing keyboard, mouse, touch and pen input

Introduction

An HTML INPUT element with a type="number" shows a spinner at the right side. This spinner is a waste of screen space if the element doesn't have the user input focus. Therefore, if not focused, numeric input controls must hide the spinners.

The spinner of a focused numeric input element can be operated with physical keyboard plus and minus buttons. On-screen keyboards lack this feature, because they don't have a key-autorepeat function.

The spinners are well suited for mouse clicks, but not for touch input with fingers. If the input element is focused with a finger-touch, an alternative UI must be shown. Audio workstation programs usually show virtual knobs for this purpose, they can be operated with a linear up/down or circular clockwise/anticlockwise gesture.

Spinners react differently to pen input. A mouse down keeps the spinner running continiously, while a pen tap produces only a single click. This must be addressed somehow. Either with an auto-repeat feature, or with knob gestures, similar to finger touch control.
